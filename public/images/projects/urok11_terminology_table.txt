┌──────────────────────────────────────────────────────────────────────────┐
│ ТЕРМИН: Критический путь                                                 │
├──────────────────────────────────────────────────────────────────────────┤
│ ОПРЕДЕЛЕНИЕ:                                                             │
│ Представьте дорогу с одним мостом. Если мост (критическая задача)       │
│ разрушен, все движение останавливается, даже если на обычных участках   │
│ дороги пробок нет. Задержка любой задачи на критическом пути напрямую   │
│ отодвигает дату завершения всего проекта.                               │
├──────────────────────────────────────────────────────────────────────────┤
│ ВАЖНОСТЬ: Самые опасные риски – те, что угрожают задачам на критическом │
│ пути. Их нужно выявлять и контролировать в первую очередь.              │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ФОРМУЛА 1: Универсальный промпт для подведения итогов                   │
├──────────────────────────────────────────────────────────────────────────┤
│ СТРУКТУРА:                                                               │
│                                                                          │
│ Ты — руководитель проектного офиса. Подведи итоги проекта:              │
│                                                                          │
│ ПРОЕКТ: [название и описание]                                           │
│ ПЕРИОД: [даты начала и окончания этапа]                                 │
│ ИСХОДНЫЕ ДАННЫЕ: [вставьте данные от разных отделов]                    │
│                                                                          │
│ ЗАДАЧИ АНАЛИЗА:                                                          │
│ • Что выполнено (перечень и %)                                          │
│ • Что не выполнено (причина)                                            │
│ • Ключевые достижения                                                   │
│ • Проблемные зоны                                                       │
│ • Общий прогресс проекта                                                │
│                                                                          │
│ ФОРМАТ ВЫВОДА:                                                           │
│ • резюме (3-4 предложения)                                              │
│ • таблица выполнения по направлениям                                    │
│ • выводы и рекомендации                                                 │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ФОРМУЛА 2: Промпт для контроля прогресса проекта                        │
├──────────────────────────────────────────────────────────────────────────┤
│ СТРУКТУРА:                                                               │
│                                                                          │
│ Ты — специалист по управлению сроками. Проанализируй график проекта:    │
│                                                                          │
│ КОНТРОЛЬНЫЕ ТОЧКИ ПРОЕКТА: [таблица с этапами и сроками]               │
│ ТЕКУЩАЯ ДАТА: [сегодня]                                                 │
│ СТАТУС ВЫПОЛНЕНИЯ: [что уже сделано]                                    │
│                                                                          │
│ ПРОАНАЛИЗИРУЙ:                                                           │
│ • Какие этапы в зоне риска срыва                                       │
│ • Как задержка одного этапа влияет на другие                           │
│ • Возможности вести работы параллельно                                 │
│ • Необходимый резерв времени                                           │
│                                                                          │
│ ВЫДАЙ:                                                                   │
│ • светофор по каждому этапу (🟢🟡🔴)                                     │
│ • график отставания/опережения                                          │
│ • прогноз окончания при текущих темпах                                 │
│ • меры по устранению отставания                                        │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ФОРМУЛА 3: Промпт для отчета "наверх"                                   │
├──────────────────────────────────────────────────────────────────────────┤
│ СТРУКТУРА:                                                               │
│                                                                          │
│ Создай резюме проекта для руководства (1 страница):                     │
│                                                                          │
│ ПРОЕКТ: [название]                                                      │
│ БЮДЖЕТ: [сумма]                                                         │
│ СРОК: [период]                                                          │
│ КОМАНДА: [количество человек]                                           │
│ ВЫПОЛНЕНО: [список достижений]                                          │
│ ПРОБЛЕМЫ: [список сложностей]                                           │
│                                                                          │
│ СТРУКТУРА РЕЗЮМЕ:                                                        │
│ 1. Заголовок с ключевой метрикой успеха                                │
│ 2. Цель и результат (было/стало)                                       │
│ 3. Ключевые достижения (3-5 пунктов)                                   │
│ 4. Преодоленные вызовы                                                 │
│ 5. Эффект для организации                                              │
│ 6. Следующие шаги                                                      │
│                                                                          │
│ ТРЕБОВАНИЯ:                                                              │
│ • Стиль: деловой, но вдохновляющий                                     │
│ • Соответствие: ГОСТ Р 7.0.97-2016, ГОСТ Р 6.30-2003                  │
│ • Объём: 1 страница А4                                                 │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ФОРМУЛА 4: Универсальный промпт для анализа рисков                      │
├──────────────────────────────────────────────────────────────────────────┤
│ СТРУКТУРА:                                                               │
│                                                                          │
│ Проведи анализ рисков проекта и выяви критические задачи:               │
│                                                                          │
│ ПРОЕКТ: [Название и краткое описание]                                   │
│ СТАДИЯ: [Текущий этап]                                                  │
│ ДЛИТЕЛЬНОСТЬ: [Общий срок или дедлайн]                                 │
│ ОСНОВНЫЕ ЗАДАЧИ: [Перечисли 5-7 ключевых задач с примерными сроками]   │
│                                                                          │
│ ВЫЯВИ РИСКИ ПО КАТЕГОРИЯМ:                                              │
│ 1. Организационные (кадры, координация)                                │
│ 2. Технические (оборудование, ПО, инфраструктура)                      │
│ 3. Финансовые (бюджет, финансирование)                                 │
│ 4. Временные (сроки, дедлайны, зависимость от других задач)            │
│ 5. Внешние (законодательство, контрагенты, согласования)               │
│                                                                          │
│ ДЛЯ КАЖДОГО РИСКА ОПРЕДЕЛИ:                                             │
│ • вероятность (высокая/средняя/низкая)                                 │
│ • влияние на сроки (критическое/существенное/незначительное)           │
│ • влияние на критический путь (Да/Нет)                                 │
│ • меры предотвращения                                                  │
│ • план действий если случится                                          │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ПРИМЕР 1: Подведение итогов проекта "Цифровизация госуслуг"            │
├──────────────────────────────────────────────────────────────────────────┤
│ ПРОМПТ:                                                                  │
│ Подведи итоги квартала по проекту цифровизации госуслуг:                │
│                                                                          │
│ ДАННЫЕ ПО НАПРАВЛЕНИЯМ:                                                  │
│                                                                          │
│ 1. Перевод услуг в электронный вид:                                    │
│    • план: 50 услуг                                                    │
│    • факт: 43 услуги                                                   │
│    • проблемы: недостаток разработчиков                                │
│                                                                          │
│ 2. Обучение сотрудников:                                                │
│    • план: 200 человек                                                 │
│    • факт: 247 человек                                                 │
│    • превышение за счет онлайн-формата                                 │
│                                                                          │
│ 3. Модернизация инфраструктуры:                                         │
│    • план: 10 рабочих мест                                             │
│    • факт: 10 рабочих мест                                             │
│    • выполнено в срок                                                  │
│                                                                          │
│ 4. Интеграция с ЕПГУ:                                                   │
│    • план: начать интеграцию                                           │
│    • факт: не начато                                                   │
│    • причина: ожидание методических указаний                           │
├──────────────────────────────────────────────────────────────────────────┤
│ РЕЗУЛЬТАТ ИИ:                                                            │
│                                                                          │
│ РЕЗЮМЕ:                                                                  │
│ Проект выполнен на 72,5% от квартального плана. Основное отставание —  │
│ по переводу услуг в электронный вид (86% выполнения) и отсутствие      │
│ начала интеграции с ЕПГУ. Компенсировано перевыполнением по обучению   │
│ персонала на 23,5%.                                                     │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ПРИМЕР 2: Автоматическое выявление критического пути                    │
├──────────────────────────────────────────────────────────────────────────┤
│ ПРОМПТ:                                                                  │
│ Проанализируй список задач и сроков. Какая последовательность задач     │
│ определяет общую длительность проекта? Выдели критический путь.         │
│                                                                          │
│ ЗАДАЧИ:                                                                  │
│ • Согласование ТЗ (10 дней)                                             │
│ • Закупка оборудования (15 дней, зависит от п.1)                       │
│ • Монтаж (20 дней, зависит от п.2)                                     │
│ • Обучение персонала (10 дней, зависит от п.3)                         │
│ • Пилотная эксплуатация (14 дней, зависит от п.4)                      │
├──────────────────────────────────────────────────────────────────────────┤
│ ОТВЕТ ИИ:                                                                │
│ Критический путь: 1→2→3→4→5 = 10+15+20+10+14 = 69 дней.                │
│ Задержка любой из этих задач сдвинет срок сдачи проекта.                │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ПРИМЕР 3: Моделирование последствий рисков                              │
├──────────────────────────────────────────────────────────────────────────┤
│ ПРОМПТ:                                                                  │
│ Смоделируй последствия риска «Задержка поставки оборудования на 7 дней» │
│ на критический путь проекта. Исходная длительность: 69 дней.            │
├──────────────────────────────────────────────────────────────────────────┤
│ ОТВЕТ ИИ:                                                                │
│ Задержка на 7 дней в задаче #2 (Закупка) приведет к сдвигу всего       │
│ критического пути: новый срок = 76 дней. Рекомендуется создать буфер   │
│ на этапе закупки или найти альтернативного поставщика.                  │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ПРИМЕР 4: Оптимизация пути при наступлении риска                        │
├──────────────────────────────────────────────────────────────────────────┤
│ ПРОМПТ:                                                                  │
│ Задача #3 (Монтаж) отстает от графика на 5 дней. Предложи 3 варианта   │
│ оптимизации критического пути чтобы сохранить общий дедлайн.            │
├──────────────────────────────────────────────────────────────────────────┤
│ ОТВЕТ ИИ:                                                                │
│ 1. Ускорение монтажа (добавить ресурсы, работать в 2 смены)            │
│ 2. Перекрытие задач (начать обучение персонала параллельно с           │
│    завершением монтажа)                                                 │
│ 3. Сокращение тестирования (уменьшить пилотную эксплуатацию с 14 до   │
│    10 дней с усиленным контролем)                                      │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐
│ ПРИМЕР 5: История успеха вместо сухих цифр                              │
├──────────────────────────────────────────────────────────────────────────┤
│ ПРОМПТ:                                                                  │
│ Перепиши сухие данные проекта как историю успеха:                       │
│                                                                          │
│ БЫЛО:                                                                    │
│ • обработка обращений: 5 дней                                           │
│ • ошибок в документах: 15%                                              │
│ • жалоб граждан: 50 в месяц                                             │
│                                                                          │
│ СТАЛО:                                                                   │
│ • обработка: 2 дня                                                      │
│ • ошибок: 3%                                                            │
│ • жалоб: 10 в месяц                                                     │
│                                                                          │
│ Напиши как историю трансформации.                                       │
│ Добавь человеческое лицо (без имени).                                   │
│ Покажи влияние на граждан.                                              │
├──────────────────────────────────────────────────────────────────────────┤
│ РЕЗУЛЬТАТ ИИ:                                                            │
│                                                                          │
│ От бумажного хаоса к цифровой эффективности                             │
│                                                                          │
│ Представьте: гражданин приходит за справкой и получает её за 2 дня     │
│ вместо привычной недели. Это реальность нашего учреждения после         │
│ внедрения СЭД.                                                           │
│                                                                          │
│ Цифры говорят сами за себя:                                             │
│ • скорость обработки увеличилась в 2.5 раза                            │
│ • количество ошибок снизилась в 5 раз                                  │
│ • удовлетворенность граждан выросла на 80%                             │
└──────────────────────────────────────────────────────────────────────────┘

┌──────────────────────────────────────────────────────────────────────────┐

│ Для любого отчета проси ИИ: "Отметь каждый пункт цветом:                │
│ 🟢 (зеленый) всё хорошо                                                  │
│ 🟡 (желтый) требует внимания                                             │
│ 🔴 (красный) критическая проблема"                                       │
│                                                                          │
│ Руководство сразу видит, где проблемы.                                  │

┌──────────────────────────────────────────────────────────────────────────┐

├──────────────────────────────────────────────────────────────────────────┤
