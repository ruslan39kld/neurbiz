КОМПОНЕНТ 1: РОЛЬ И СПЕЦИАЛИЗАЦИЯ
-----------------------------------
Кто ты как ассистент, в какой сфере эксперт, какой уровень знаний используешь

Примеры:
├─ Ты — специалист отдела по обращениям граждан в администрации
├─ Ты — AI-ассистент HR-аналитика, разбирающийся в KPI, OKR, динамике персонала
└─ Ты — эксперт по налоговому консультированию для ИП и самозанятых

КОМПОНЕНТ 2: ЗАДАЧА
-----------------------------------
Что конкретно ты должен делать, какие запросы обрабатывать

Пример: Твоя задача — составлять официальные ответы на обращения граждан в деловом стиле, с учётом закона 59-ФЗ и регламентов ведомства

КОМПОНЕНТ 3: СТИЛЬ И ТОН
-----------------------------------
Каким должен быть стиль речи, насколько формально/дружелюбно/точно писать

Параметры:
├─ Стиль: официальный / деловой / нейтральный / дружелюбный / технический
├─ Тон: вежливый / формальный / понятный / строгий / позитивный
└─ Язык: русский / английский / билингва

КОМПОНЕНТ 4: СТРУКТУРА ОТВЕТА
-----------------------------------
Четкий порядок блоков в каждом ответе

Типовая структура:
├─ Приветствие к адресату
├─ Суть обращения / запроса
├─ Правовое или логическое обоснование
├─ Действия, сроки и результат
└─ Контактные данные для уточнений

КОМПОНЕНТ 5: ОСОБЫЕ УСЛОВИЯ
-----------------------------------
Ссылки на законы, сроки ответов, внутренние правила, формат, стиль

Примеры:
├─ Соблюдать закон 59-ФЗ
├─ При переадресации — обязательно обосновывать
└─ Соответствие требованиям ГОСТ Р 7.0.97-2016

КОМПОНЕНТ 6: КОНТЕКСТ ПРИМЕНЕНИЯ
-----------------------------------
Где и кем используется ассистент

Пример: Используется сотрудниками администрации при подготовке писем в официальных системах документооборота

КОМПОНЕНТ 7: ПРИВЕТСТВЕННОЕ СООБЩЕНИЕ
-----------------------------------
Что ассистент пишет первым при старте диалога

Пример: Здравствуйте! Я помогу вам составить официальный ответ на обращение гражданина. Напишите суть обращения, ФИО и дату — и я подготовлю текст по форме.

КОМПОНЕНТ 8: ЧТО НЕЛЬЗЯ
-----------------------------------
Категорические запреты

Строго запрещается:
├─ Генерировать от имени пользователя или заявителя
├─ Выдумывать факты, даты, названия организаций или законы
├─ Использовать субъективные мнения или эмоциональные оценки
├─ Писать в неформальном, ироничном или разговорном стиле
├─ Ссылаться на несуществующие документы
├─ Генерировать текст вне компетенции
└─ Искажать структуру официального ответа или допускать двусмысленности

ТИПЫ АССИСТЕНТОВ ДЛЯ ГОССЕКТОРА
=============================================================================

ТИП 1: АССИСТЕНТ ДЛЯ ДОКУМЕНТООБОРОТА
-----------------------------------
Специализация: помощь в составлении служебных документов

Документы:
├─ Служебные записки и письма
├─ Ответы на обращения граждан
├─ Протоколы совещаний
├─ Приказы и распоряжения
└─ Информационные справки

Стиль работы:
├─ Строгое соблюдение делового стиля
├─ Соответствие требованиям ГОСТ Р 7.0.97-2016
├─ Грамотное использование канцеляризмов
└─ Четкая структура документов

Обязательные вопросы:
1. Тип документа
2. Адресат (БЕЗ УКАЗАНИЯ ФИО)
3. Суть обращения (краткое описание темы)
4. Юридическое обоснование (ссылки на НПА)
5. Конкретные действия и сроки

ТИП 2: АССИСТЕНТ-АНАЛИТИК
-----------------------------------
Специализация: подготовка аналитических материалов

Экспертные области:
├─ Социально-экономическая статистика
├─ Анализ эффективности госпрограмм
├─ Мониторинг показателей развития
└─ Сравнительный анализ регионов

Стандартная структура анализа:
1. Executive Summary (2-3 ключевых вывода)
2. Исходные данные и методология
3. Детальный анализ по направлениям
4. Сравнение с планами/нормативами
5. Выявленные риски и возможности
6. Конкретные рекомендации
7. План мониторинга

ТИП 3: АССИСТЕНТ ДЛЯ КОММУНИКАЦИЙ
-----------------------------------
Специализация: подготовка материалов для внешних коммуникаций

Виды материалов:
├─ Пресс-релизы и новости
├─ Посты для социальных сетей
├─ Разъяснительные материалы для граждан
└─ Ответы на запросы СМИ

Принципы работы:
├─ Простой понятный язык без канцеляризмов
├─ Фокус на пользе для граждан
├─ Позитивная подача информации
└─ Конкретные факты и цифры

Адаптация под аудитории:
├─ Пенсионеры: простые термины, пошаговые инструкции
├─ Молодежь: современный язык, акцент на удобстве
└─ Бизнес: фокус на возможностях и выгодах

ПЛАТФОРМЫ ДЛЯ СОЗДАНИЯ AI-АССИСТЕНТОВ
=============================================================================

ЯНДЕКС НЕЙРОЭКСПЕРТ
-----------------------------------
Подходит для работы с документами

DEEPSEEK (работа в одном треде)
-----------------------------------
Способ: задание начальных инструкций в первом сообщении треда

Как работает:
├─ В начале разговора даете подробные инструкции
├─ ИИ "запоминает" их на время сессии
└─ Все последующие сообщения выполняются в рамках этой роли

ПРЕИМУЩЕСТВА:
✓ Работает в любой ИИ-модели
✓ Не требует специальных функций платформы
✓ Бесплатно и доступно везде

НЕДОСТАТКИ:
✗ Нужно повторять инструкции в новых сессиях
✗ Может "забыть" роль в длинных разговорах
✗ Менее стабильно, чем специализированные ассистенты

Роль: эксперт по нормативно-правовым актам

Структура ответа:

ШАБЛОН 2: АНАЛИТИК ПОКАЗАТЕЛЕЙ
-----------------------------------
Роль: специалист по мониторингу KPI
Задача: анализ выполнения плановых показателей

Структура ответа:
├─ Динамика показателей
├─ Отклонения от плана
├─ Причины
└─ Прогноз

ШАБЛОН 3: ПЕРЕВОДЧИК С "ЧИНОВНИЧЬЕГО"
-----------------------------------
Роль: редактор текстов для граждан
Задача: адаптация сложных текстов в понятный язык

Структура ответа:
├─ Упрощённый текст
├─ Ключевые термины
└─ Пошаговая инструкция

ЛАЙФХАКИ ДЛЯ СОЗДАНИЯ ЭФФЕКТИВНЫХ АССИСТЕНТОВ
=============================================================================

#1: Начните с реальных примеров
└─ Загрузите в ассистента 3-5 примеров ваших лучших документов

Инструкция: "Изучи эти примеры моих ответов и используй их стиль и структуру для новых задач"

#2: Создайте чек-лист для ассистента
└─ Встройте в инструкции проверочный список

Проверка перед финализацией:
☐ Все ли реквизиты указаны?
☐ Соблюдены ли сроки?
☐ Есть ли ссылки на НПА?
☐ Понятен ли язык целевой аудитории?
☐ Указаны ли контактные данные?

#3: Обучите ассистента задавать вопросы
└─ Научите уточнять недостающую информацию

Уточняющие вопросы:
├─ Кто адресат документа?
├─ Какие конкретно действия требуются?
├─ Есть ли особые требования к срокам?
└─ Нужны ли дополнительные согласования?

#4: Создайте специализированных помощников
└─ Лучше 3 узкоспециализированных ассистента, чем 1 универсальный

Пример набора:
1. Ассистент по письмам гражданам
2. Ассистент по аналитическим справкам
3. Ассистент по презентациям для руководства

#5: Регулярно обновляйте инструкции
└─ Каждый месяц анализируйте работу ассистента

Вопросы для анализа:
├─ Какие ошибки повторяются?
├─ Что можно улучшить в инструкциях?
└─ Какие новые требования появились?

КЛЮЧЕВЫЕ ВЫВОДЫ УРОКА 22
=============================================================================

1. AI-ассистент – это специально настроенная версия ИИ с заранее прописанными инструкциями, работающая как постоянный помощник

2. Главное преимущество – не нужно каждый раз объяснять роль и требования, что экономит время и обеспечивает консистентность

3. Базовая структура инструкции включает 8 компонентов: роль, задача, стиль, структура ответа, особые условия, контекст, приветствие, запреты

4. Для госсектора наиболее полезны 3 типа ассистентов: для документооборота, аналитики и коммуникаций

5. Лучше создать 3 узкоспециализированных ассистента, чем 1 универсальный

6. Важно загрузить в ассистента реальные примеры ваших лучших документов для обучения на образцах

7. Ассистент должен уметь задавать уточняющие вопросы, если информации недостаточно

8. Необходимо регулярно обновлять инструкции на основе анализа работы и новых требований

9. Обязательно прописывать четкие запреты (что делать нельзя) для избежания ошибок

10. Чек-лист создания включает 4 этапа: планирование, создание инструкций, тестирование, внедрение и поддержка
